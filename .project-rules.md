# 📁 Copilot Prompts 项目文件组织规范

> 为保持项目整洁，所有新生成的文档必须遵循以下规范

## 📂 目录结构规范

### 根目录（仅保留核心文档）
```
copilot-prompts/
├── README.md                 # 项目主文档
├── CHANGELOG.md              # 版本变更日志
├── QUICK_START.md            # 快速开始指南
├── STRUCTURE.md              # 项目结构说明
├── .project-rules.md         # 本文件：项目规范
└── archive/                  # 归档文件夹
    └── reports/              # 历史报告
        ├── CLEANUP_V2_REPORT.md
        └── RULES_REORGANIZATION.md
```

### 专用目录
- **`mcp-server/`** - MCP 服务器代码
  - `src/` - 源代码
  - `test-*.cjs` - 测试文件
  - `package.json`, `tsconfig.json` - 配置文件

- **`standards/`** - MCP 编码规范库
  - `core/` - 核心规范
  - `frameworks/` - 框架规范
  - `libraries/` - 库规范
  - `patterns/` - 模式规范

- **`docs/`** - 项目文档
  - 技术文档
  - API 文档
  - 设计文档

- **`archive/`** - 归档文件
  - `reports/` - 历史报告（已完成的任务报告、分析报告等）
  - `deprecated/` - 已废弃的文件

## 🚫 禁止行为

❌ **不要在根目录创建以下类型的文件：**
- 任务报告（如 `*_REPORT.md`, `*_COMPLETE.md`）
- 实现计划（如 `*_PLAN.md`, `*_IMPLEMENTATION.md`）
- 临时测试文件
- 个人笔记

## ✅ 文件放置规则

### 新报告文件
```bash
# ✅ 正确：放入 archive/reports/
archive/reports/ELEMENT_PLUS_ANALYSIS_2025-12-15.md

# ❌ 错误：不要放在根目录
ELEMENT_PLUS_ANALYSIS.md
```

### 新文档
```bash
# ✅ 正确：放入 docs/
docs/mcp-usage-guide.md

# ❌ 错误：不要放在根目录
MCP_USAGE_GUIDE.md
```

### 测试文件
```bash
# ✅ 正确：放入对应的模块目录
mcp-server/test-phase4.cjs

# ❌ 错误：不要放在根目录
test-something.js
```

## 📝 文件命名规范

### 报告文件
- 格式：`{主题}_{日期}.md`
- 示例：`ELEMENT_PLUS_ANALYSIS_2025-12-15.md`
- 位置：`archive/reports/`

### 文档文件
- 格式：`{描述性名称}.md`
- 示例：`mcp-usage-guide.md`
- 位置：`docs/`

### 配置文件
- 保持在对应模块的根目录
- 示例：`mcp-server/package.json`

## 🔄 整理清理流程

### 定期检查（每次对话结束）
1. 扫描根目录是否有新增的报告/临时文件
2. 将报告移动到 `archive/reports/`
3. 将文档移动到 `docs/`
4. 删除临时文件

### 执行命令
```bash
# 移动报告文件
mv *_REPORT.md archive/reports/ 2>/dev/null
mv *_COMPLETE.md archive/reports/ 2>/dev/null
mv *_PLAN.md archive/reports/ 2>/dev/null

# 列出根目录文件（检查是否整洁）
ls -la | grep -E '\.md$' | grep -v README | grep -v CHANGELOG
```

## 📌 AI 指令

**对 AI 的要求：**
1. ✅ 生成任何报告时，直接创建在 `archive/reports/` 目录
2. ✅ 生成任何文档时，直接创建在 `docs/` 目录
3. ✅ 不要在根目录创建新的 `.md` 文件（除非是核心文档的更新）
4. ✅ 每次对话结束前，主动检查并整理根目录
